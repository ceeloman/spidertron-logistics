Version: 3.2.0
  Features:
    - Added manual dump button in GUI: players can now manually trigger item dumping for spidertrons
    - Enhanced spidertron activation and deactivation: improved logic for GUI switch state handling with better responsiveness
    - Improved item dumping checks: spiders now check for dumpable items during activation and efficiently manage inventory
    - Multi-route assignment: spiders can now handle multi-item and multi-requester deliveries more efficiently
  Pathfinding & Terrain:
    - Enhanced water gap tolerance: spiders can now navigate over small water gaps by detecting crossings and finding closest land points
    - Improved waypoint adjustments: better handling of water crossings that exceed spider's stepping capability
    - Enhanced waypoint smoothing: prevents smoothing over water when spiders cannot traverse it, ensuring more accurate pathing
    - Improved pathfinding with pre-validated waypoints: more efficient navigation to requesters using validated waypoint data
    - Enhanced pathfinding safety: improved checks for detours around enemy nests and water crossings
  Logistics & Assignment:
    - Enhanced beacon assignment: context-aware parameters for nearest beacon search and assignment functions
    - Improved beacon finding: new function to find beacons with highest pickup count for better assignment logic
    - Streamlined network handling: changed from beacon-based to surface-wide network management
    - Enhanced item management: improved helper function to retrieve logistic requests and better handling of dumpable items
    - Improved journey management: better item handling during pickups and deliveries with refined dumpable item processing
  Performance:
    - Optimized path data cleanup: stored path data is now cleaned up after use
  Bugfixes:
    - Fixed item dumping logic to avoid dumping requested items
    - Improved beacon assignment robustness during entity lifecycle events

Version: 3.1.0
  Features:
    - Added automatic item dumping: spiders now automatically dump excess items (items not needed for their logistic requests) into storage chests
    - Added stuck detection: spiders that haven't moved for 10+ seconds will automatically attempt to repath
    - Enhanced pathfinding with waypoint simplification: adaptive spacing based on terrain difficulty and path curvature for more efficient routing
    - Improved pathfinding with waypoint smoothing: corner cutting for smoother, more natural movement paths
    - Enhanced nest avoidance: improved detour calculation that chooses the shorter route around enemy nests
  Pathfinding & Terrain:
    - Improved waypoint filtering: better detection and filtering of water/cliff waypoints for non-capable spiders
    - Enhanced pathfinding error handling: visual error messages when pathfinding fails
    - Improved pathfinding retry logic: better handling of "try_again_later" responses from pathfinding API
  Logistics & Assignment:
    - Added periodic beacon validation: automatic checks every 10 seconds to detect and reassign destroyed beacons
    - Improved item dumping logic: spiders check for dumpable items after failed deliveries and automatically route to storage chests
  Bugfixes:
    - Fixed issue where spiders could get stuck with items they don't need after failed deliveries
    - Improved handling of spiders with excess inventory items
    - Fixed pathfinding issues when chunks are not yet generated
  Other:
    - Added visual error feedback for pathfinding failures

Version: 3.0.0
  Features:
    - Changed requester and provider chest recipes to use basic materials (steel-plate, electronic-circuit, advanced-circuit) instead of requester-chest/storage-chest and spidertron-remote
    - Removed spidertron-remote requirement from logistic beacon recipe
    - Moved logistic beacon recipe unlock to advanced-electronics technology
    - Removed "logistic-system" prerequisite from spidertron-logistic-system technology
    - Activate Spider logistics with a toggle in their inventory
  Compatibility:
    - Added dynamic detection of first spider vehicle technology for better mod compatibility
    - Removed hardcoded mod compatibility code for Insectitron and spidertrontiers-community-updates (now handled dynamically)
      - Technology should be available to unluck after advanced-electronics and the earliest Spider-vehicle unlock
  Code Structure:
    - Major refactor: modularized codebase into lib/ directory with separate modules for beacon assignment, GUI, journey management, logistics, pathfinding, registration, rendering, terrain, and utilities
    - Split data.lua into separate files: data/entities.lua, data/items.lua, data/recipes.lua, data/technology.lua
    - Improved code organization and maintainability
  Pathfinding & Terrain:
    - Added intelligent pathfinding system with water traversal detection based on spider leg collision masks
    - Implemented cliff detection and avoidance for small spiders (hopefully)
    - Added terrain awareness to avoid dangerous areas (enemy nests & water for non-water-capable spiders)
    - Enhanced pathfinding to handle long-distance travel with waypoint system
  GUI & User Experience:
    - Improved requester chest GUI with multi-item request support
    - Added item selector modal with quantity slider
    - Added spidertron toggle button in GUI to activate/deactivate logistics
  Logistics & Assignment:
    - Added robot logistic chests (storage-chest, active-provider-chest, passive-provider-chest) as providers
    - Improved beacon assignment system with validity checks and automatic reassignment
    - Enhanced item management with retry logic for pickup and dropoff operations
    - Added incoming items tracking to prevent over-allocation
    - Improved spider assignment algorithm to consider water traversal capabilities
  Performance:
    - Optimized logging system (commented out excessive debug logs for better performance)
    - Improved update cycles and reduced unnecessary processing
  Graphics:
    - Updated spidertron logistic beacon graphics and icons
    - Adjusted collision and selection boxes for better usability
    - Enhanced animation settings
  Bugfixes:
    - Fixed beacon assignment issues when beacons are destroyed
    - Improved handling of invalid entities and edge cases
    - Added retry logic for failed item pickups and dropoffs
    - Fixed GUI issues with item quantity display
  Other:
    - Added comprehensive logging system (lib/logging.lua)
    - Added debug commands for spider management
    - Added PROJECT_PLAN.md documentation
    - Improved error handling and validation throughout
